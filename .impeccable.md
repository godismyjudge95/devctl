## Design Context

### Users
Solo PHP developers and small teams on Linux. Daily active users — this dashboard is open in a browser tab all day. Technical, comfortable with CLI and sysadmin concepts. Primary jobs: check service health at a glance, restart/stop services, manage local PHP virtual hosts, inspect variable dumps and test emails. Occasional jobs: install new services, adjust settings, tail logs.

### Brand Personality
Confident. Precise. Unapologetically technical. Think: a well-maintained terminal, not a SaaS dashboard. devctl doesn't need to explain itself with marketing copy — the user already knows what it is. Three words: **sharp, utilitarian, reliable**.

### Aesthetic Direction
Technical character — lean into the developer/CLI heritage without going full skeuomorphic terminal. Visual language should feel like it belongs in the same world as VS Code, TablePlus, and `htop`. Monospace accents for data (versions, paths, credentials, ports). High information density in tables — not spacious SaaS padding. Font: Geist (developer-designed, distinctive). Restrained indigo primary palette, current OKLCH tokens are good.

Anti-references: Vercel-style "everything is a gradient", generic shadcn-vue-default, anything that looks like a SaaS onboarding wizard.

### Design Principles
1. **Data over decoration** — every visual element must carry information or aid navigation. Badges that label the default state (e.g. "Auto" on 33/33 rows) are noise.
2. **Consistent affordances** — if one page uses labeled buttons, all pages do. Same action = same pattern, always.
3. **Silence is not feedback** — any user action that changes state must produce visible confirmation (toast, inline indicator). Silent auto-save is a UX bug.
4. **Density is respectful** — users are professionals who can read dense tables. Don't pad to hide the lack of design; strip redundant content instead.
5. **Monospace has meaning** — use `font-mono` for values (versions, paths, credentials, ports, timestamps). Use prose fonts for labels and descriptions.
